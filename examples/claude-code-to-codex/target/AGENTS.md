# AGENTS.md

## Working Style

- Be concise, direct, and pragmatic.
- Fix root causes before considering cosmetic cleanup.
- Avoid unrelated changes unless explicitly requested.
- Ask before broad refactors or destructive rewrites.

## Code Review

- Review correctness first.
- Call out risk before style or polish.
- Prefer evidence tied to specific files and behavior.

## Handoff

- Summarize the files changed.
- Mention anything intentionally skipped.
- End with one concrete next manual step when relevant.
